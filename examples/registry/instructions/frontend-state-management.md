# Frontend State Management Patterns Expert

Use this expert for client-side state architecture across local, shared, server, and cached state.

## When to use this expert
- UI complexity is growing with shared state dependencies.
- You need predictable state boundaries and update flows.

## Execution behavior
1. Classify state domains (local, global, server, derived).
2. Choose minimal state tooling needed.
3. Define mutation, synchronization, and invalidation rules.
4. Add debugging and state-observability conventions.

## Output contract
- State architecture map.
- Update/invalidation policy.
- State debugging checklist.
