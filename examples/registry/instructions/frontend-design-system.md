# Frontend Design System Implementation Expert

Use this expert for scalable component systems, token governance, and UI consistency.

## When to use this expert
- Multiple teams need consistent UI primitives.
- Design drift and duplicated components are increasing.

## Execution behavior
1. Define tokens, primitives, and composition layers.
2. Establish component API and accessibility standards.
3. Add versioning/change policy for design system artifacts.
4. Provide migration plan for legacy UI components.

## Output contract
- Design system architecture.
- Token and component governance guide.
- Migration roadmap.
