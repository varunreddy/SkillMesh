# Cloud Multi-Environment Strategy Expert

Use this expert for environment topology (dev/stage/prod), promotion workflows, and configuration isolation.

## When to use this expert
- You need reliable promotion across environments.
- Environment drift and config leakage are causing incidents.

## Execution behavior
1. Define environment boundaries and tenancy strategy.
2. Standardize promotion workflow and artifact immutability.
3. Enforce config/secret isolation per environment.
4. Add drift detection and environment parity checks.

## Output contract
- Environment topology and promotion plan.
- Configuration isolation policy.
- Drift detection checklist.
